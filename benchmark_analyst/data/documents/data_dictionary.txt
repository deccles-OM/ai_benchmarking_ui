DATA DICTIONARY
Ford Analytics Data Model
Version: 1.5

TABLE: warranty_claims
Description: Detailed warranty claim records for all Ford vehicles

COLUMNS:
claim_id (INT)
  - Technical ID: Unique claim identifier
  - Business Logic: System-generated sequential ID
  - Valid Range: Positive integers
  - Usage: Primary key

model (STRING)
  - Description: Vehicle model name
  - Valid Values: F150, Escape, Bronco, Mustang
  - Business Rule: Must match vehicle master data
  - Null Policy: NOT NULL

mileage (INT)
  - Description: Vehicle mileage at time of claim in miles
  - Valid Range: 0 to 300,000 miles
  - Business Rule: Must be >= last recorded mileage
  - Null Policy: Can be NULL if not recorded; required for warranty validation
  - Note: Missing values should be imputed with regional average

repair_cost (DECIMAL(10,2))
  - Description: Total warranty repair cost in USD
  - Valid Range: > 0 (must be positive)
  - Business Rule: Cannot exceed policy limits; must be approved by dealer
  - Null Policy: NOT NULL
  - Note: Negative values indicate credits/adjustments - should be flagged as data errors

failure_code (STRING)
  - Description: OBD-II or Ford-specific diagnostic code
  - Valid Values: P0xxx for emissions, C0xxx for chassis, U0xxx for network, B0xxx for body
  - Business Rule: Cross-reference with Ford Technical Service Bulletins
  - Null Policy: Can be NULL for non-diagnostic repairs

claim_date (DATE)
  - Description: Date claim was submitted
  - Format: YYYY-MM-DD (UTC)
  - Business Rule: Must be <= current date
  - Null Policy: NOT NULL

dealer_id (STRING)
  - Description: Ford dealership code
  - Format: D{4 digits} (example: D0042)
  - Business Rule: Must exist in dealer master table
  - Null Policy: NOT NULL

region (STRING)
  - Description: Geographic region
  - Valid Values: US, EU, Asia
  - Business Rule: Derived from dealer location
  - Null Policy: NOT NULL

TABLE: sales_data
Description: Vehicle sales by model, region, and time period

COLUMNS:
model (STRING)
  - Valid Values: F150, Escape, Bronco, Mustang
  - Null Policy: NOT NULL

region (STRING)
  - Valid Values: US, EU, Asia
  - Null Policy: NOT NULL

units_sold (INT)
  - Description: Number of units sold
  - Valid Range: >= 0
  - Business Rule: Monthly/regional aggregate
  - Null Policy: NOT NULL

year (INT)
  - Valid Range: 2020 to current year
  - Null Policy: NOT NULL

TABLE: dealer_claims
Description: Aggregated warranty claims by dealership

dealer_id (STRING)
  - Format: D{4 digits}
  - Null Policy: NOT NULL

claims_per_month (INT)
  - Valid Range: >= 0
  - Business Rule: Should align with detailed claims when aggregated
  - Null Policy: NOT NULL

region (STRING)
  - Valid Values: US, EU, Asia
  - Null Policy: NOT NULL
